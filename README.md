# Каталог товаров - Мини-приложение

Проект содержит две версии приложения "Каталог товаров" с разной реализацией отображения списка товаров. Основной функционал соответствует требованиям: 3 экрана с интеграцией API, локальным кешированием, offline-режимом и модульной архитектурой.

## Ветки проекта

### Ветка `master` (наследуется от `paging`)
- [x] **Использует Paging 3** для пагинации списка товаров
- Реализует эффективную загрузку данных по мере прокрутки

### Ветка `no-paging`
- [ ] **Без Paging 3** - классическая загрузка списка
- Однократная загрузка всех товаров при открытии экрана
- Более простая реализация

## Функционал приложения

### 1. Экран списка товаров
- Загрузка данных из API (FakeStoreAPI)
- Вывод списка с картинкой, заголовком и ценой
- Фильтрация по категориям (данные из API)
- Локальный поиск по заголовку
- Offline-режим: показ кешированных данных при отсутствии сети

### 2. Экран карточки товара
- Детальная информация о товаре
- Фотография товара
- Полное описание
- Рейтинг товара
- Кнопка "Добавить в избранное" с сохранением в Room

### 3. Экран избранного
- Отдельный экран с сохраненными товарами
- Данные хранятся в локальной БД Room
- Управление списком избранного
- Доступен в offline-режиме

### Паттерны и подходы
- **MVVM + MVI** для управления состоянием UI
- **Clean Architecture** с четким разделением слоев
- **Repository Pattern** с интерфейсами и реализациями
- **Dependency Injection** через Hilt

## Стек технологий

### Основные технологии
- **Kotlin** - основной язык разработки
- **Jetpack Compose** - декларативный UI фреймворк
- **Compose Navigation** - навигация между экранами
- **Coroutines & Flow** - асинхронные операции и потоки данных
- **Room** - локальная база данных для избранного и кеширования
- **Retrofit + OkHttp** - сетевые запросы и HTTP клиент
- **Coil 3** - загрузка и кеширование изображений
- **Dagger Hilt** - dependency injection

### Модульность и DI
- **AssistedInject** для фабрик зависимостей
- **Модульная структура** с разделением ответственности

### Управление состоянием и данные
- **Sealed классы** для обработки состояний (Loading, Success, Error)
- **Result wrapper** для унифицированной обработки результатов
- **Offline-first подход** - приоритет кешированных данных

### UI и навигация
- **BottomNavigation** для навигации между основными разделами
- **Compose Navigation** с type-safe подходами
- **Обработка edge-кейсов**: пустые списки, retry, онлайн/оффлайн переключения
